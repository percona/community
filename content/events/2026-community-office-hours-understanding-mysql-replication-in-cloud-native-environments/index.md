## Join the Community Office Hours

This session focuses on MySQL built on Percona Server for MySQL with native MySQL Group Replication on Kubernetes. It’s a significant step for the MySQL and Cloud Native community, offering more open-source options for deploying reliable, consistent MySQL clusters at scale.

If you’re a MySQL engineer, DevOps practitioner, SRE, or someone passionate about Kubernetes, you’ll find this session valuable.

## What’s This Session About?

We’ll walk through the release and what it means for the community, with a focus on replication models, high availability, and practical guidance for running MySQL on Kubernetes.

## Key Topics We’ll Cover

- What MySQL Group Replication looks like in a cloud-native environment  
- How the Percona Operator for MySQL enables synchronous and asynchronous replication  
- Differences between replication models and when to use each  
- How this complements the existing Percona XtraDB Cluster Operator (PXC)  
- Practical considerations for MySQL high availability on Kubernetes  
- What the new GA release brings for the community  

## Speakers

- **Marco Tusa**, MySQL Technical Leader, Percona  
- **Ege Güneş**, Senior Software Engineer, Cloud Team, Percona  

Both speakers worked closely on the technology behind the new MySQL Operator with Group Replication and will share insights from engineering, real-world testing, and operator design.
