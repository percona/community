pg_tde — Transparent Data Encryption

Encrypt your PostgreSQL data at rest without changing a single line of application code. pg_tde works at the storage level, covering both data files and WAL records.

Designed for organizations with compliance requirements (HIPAA, PCI DSS, GDPR) who can't afford to modify their application layer.

  • 🔐 File-level encryption, transparent to applications
  • 🗝️ Integrates with external KMS and keyring files
  • 📜 Encrypts both data files and WAL records
  • ✅ Supports PostgreSQL 17 and 18
Documentation GitHub

pg_oidc_validator — OIDC Authentication

Replace password-based PostgreSQL authentication with your organization's existing identity provider. pg_oidc_validator validates JWTs issued by any OIDC-compliant provider — no separate password management needed.

Works with Keycloak, Microsoft Entra ID, and any other OIDC-compliant provider. Integrates with OAuth 2 device flow via libpq.

  • 🔑 JWT validation via OIDC providers (Keycloak, Entra ID, and more)
  • 🔄 OAuth 2 device flow support through libpq
  • ⚙️ Flexible identity mapping via configurable JWT fields
  • ✅ Supports PostgreSQL 18
GitHub

pg_stat_monitor — Query Performance Analytics

A drop-in replacement for pg_stat_statements that goes much further — grouping statistics into configurable time buckets and capturing actual query plans and parameter values.

Stop guessing which queries caused a performance spike. With time-based bucketing you get pinpoint visibility into exactly when and why slowdowns occurred.

Pair it with Percona Monitoring and Management (PMM) for a complete observability solution — query analytics, dashboards, alerting, and slow query visualization all in one place.

  • 📊 Time-bucket metrics — per-interval stats, not just lifetime averages
  • 🔍 Captures actual parameter values for easier debugging
  • 📋 Displays real execution plans alongside statistics
  • 📈 Deep PMM integration — dashboards, QAN, and alerting out of the box

Percona Operator for PostgreSQL

Running PostgreSQL on Kubernetes? The Percona Operator automates everything — provisioning, scaling, backups, TLS, failover, and upgrades — applying industry best practices consistently across your clusters.

Built for teams who want the reliability of a managed database without handing control to a cloud vendor.

  • ☸️ Full lifecycle automation on any Kubernetes
  • 🔄 Automated failover and rolling upgrades
  • 💾 Scheduled backups with point-in-time recovery
  • 📈 Native PMM monitoring integration
